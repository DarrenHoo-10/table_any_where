# 炸金花牌桌自由视角设计

## 目标

让 `zha_room` 3D 牌桌支持围绕牌桌的自由观察，同时保留现有牌桌点击、下注按钮和玩家座位逻辑。

## 方案

采用轨道相机（orbit camera）：相机围绕牌桌中心旋转，而不是继续只修改第一人称的视线偏移。

- 拖拽牌桌画布：修改水平环绕角和垂直俯仰角。
- 滚轮：修改相机与观察中心的距离。
- 水平角不设硬边界，垂直角和距离设合理上下限，避免翻转、穿过桌面或穿墙。
- 牌局状态切换时继续使用现有的相机过渡；用户拖拽后保留当前自由视角，不被每帧重置。
- 点击事件继续使用现有的 `handleSceneClick`，只有发生明显拖拽时才取消点击。

## 数据流

`bindEvents()` 记录拖拽起点和当前轨道参数；`updateZhaCamera()` 根据相机模式、轨道角度和距离计算位置，并继续让相机朝向牌桌中心。炸金花房间的 `cameraMode` 仍可由房间状态驱动，但不再使用第一人称专用的 `lookYawOffset` / `lookPitchOffset` 作为唯一视角控制。

## 边界与兼容

- 旧的经典牌桌继续使用现有 `yaw`、`pitch`、`distance` 控制。
- 炸金花房间的默认相机仍从当前 overview / first-person 过渡位置开始。
- 拖拽阈值继续保留，避免点击桌面时误触发旋转。
- 不新增依赖，不改变 WebSocket、牌局状态或座位坐标。

## 验证

- 添加客户端回归测试，确认 zha 场景绑定轨道相机参数、拖拽和滚轮控制，并保留点击拖拽阈值。
- 运行 `npm run check` 和 `npm test`。
- 使用本地浏览器创建房间，验证拖拽旋转、滚轮缩放、桌面点击和下注按钮均可用，页面无新的 console error/warn。
